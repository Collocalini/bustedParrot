Планы по развитию сайта
Сайт Neocities, на котором я держу свой сайт(Противопопугайный сайт), содержит зеркало моего сайта в IPFS. Однако, я создал независимое зеркало сайта в IPFS, из-за неадекватной, на данный момент, реализации функций относящихся к IPFS на Neocities.

Site future development plans
The Neocities, which hosts my site(Never Parrot), already keeps the IPFS mirror of my site. However I've created the separate one because the current realization of IPFS related functions on Neocities is too raw and glitchy.
Ограничения, накладываемые Neocities на бесплатно хостящиеся сайты, при переходе в IPFS становятся неактуальны. Например, появляется возможность хранить большие файлы(видео, архивы и т.д.) непосредственно на сайте, не прибегая к услугам стороннего хранилища(Яндекс.Диск).
Также, по мере достижения предела в 100МБ на размер сайта, постепенно, некоторые файлы, уже размещённые на Neocities, будут переноситься в IPFS для экономии места. В первую очередь будут уходить более старые или менее важные файлы. К файлам я отношу в том числе и картинки.
Для доступа к файлам в IPFS, в настоящий момент требуется установка спецального ПО, которую обещают сделать необязательной в будущем, переписав ПО на JavaScript, что позволит запускать его непосредственно в браузере. Сылки на перенесённые в IPFS файлы, в первое время будут идти через публичные шлюзы (на пример ipfs.io), а затем перенесены на локальные(127.0.0.1:8080/ipfs/...). Иными словами, в первое время после переноса, доступность файлов людям без спецального ПО будет обусловлена только физической доступностью файлов на публичных шлюзах, а затем прямых ссылок через публичные шлюзы не останется, и для доступа к ним нужно будет установить ПО. Что я желаю всем сделать как можно раньше, поддержав распределённую сеть, и этот сайт.
Если вы уже установили ПО IPFS, у вас есть возможность не дожидаться, когда ссылки поменяются с более медленных публичных шлюзов на локальные, а воспользоваться дополнением к браузеру IPFS Gateway Redirect, которое поменяет их автоматически.

The limitations that Neocities puts on free hosted sites become unnecessary when one moves to IPFS. For example it's possible to store big files(videos, archives, etc) directly on site, instead of using services of some separate storage(Yandex.Disk).
Also, as my site reaches the 100MB size limit, I will gradually move some files stored on Neocities at the moment to IPFS to save space. Old or unimportant files will be moved first. And by files I mean pictures as well.
At the moment the special software is required to be installed in the system to access files in IPFS. This is promised to change in the future as JavaScript version of software will be available, which will allow to run IPFS directly in browser. At first I will use links that reference files through public gateways(i.e. ipfs.io) and then switch them to local gateways(127.0.0.1:8080/ipfs/...). In other words at first the moved files will be available to people without special software as long as gates can serve them, but later I won't use links referencing files through public gateways and special software will become mandatory. And I encourage everyone to install it as soon as possible to support distributed network and this site.
If you already have IPFS software installed you don't have to wait for me to switch links from the slow public gateways to the local ones. Instead you can use browser add-on IPFS Gateway Redirect to switch them for you.
